# ADR-003: SQLite como motor de base de datos

## Estado
Aceptado

## Contexto
El proyecto .NET usa SQLite con EF Core. Necesitamos la misma BD en Harbour.

## Decisión
Usar SQLite3 via contrib `hbsqlit3`. Crear las tablas manualmente en el primer arranque (no hay EnsureCreated ni migraciones). El esquema debe ser idéntico al proyecto .NET para compatibilidad de BD.

## Consecuencias
Positivas:
- Misma BD que el proyecto .NET — se pueden intercambiar
- Sin servidor, sin configuración
- hbsqlit3 es maduro y rápido

Negativas:
- Sin ORM — SQL directo (pero Harbour tiene buen soporte SQL)
- Schema drift: cambios de esquema requieren ALTER TABLE manual
- decimal se almacena como TEXT (lo mismo que en .NET con EF Core + SQLite)
