# ADR-005: SOAP AEAT con libcurl (hbcurl)

## Estado
Aceptado

## Contexto
VERI*FACTU requiere envío de registros de facturación a la AEAT vía SOAP/XML sobre HTTPS con certificado cliente.

## Decisión
Usar `hbcurl` (contrib que envuelve libcurl) para las peticiones HTTP. El XML SOAP se construye con `hbmxml` o cadenas. El certificado cliente (FNMT) se pasa a libcurl via opciones CURLOPT_SSLCERT.

## Consecuencias
Positivas:
- libcurl es la librería HTTP más completa y probada
- Soporta HTTPS, certificados cliente, timeouts
- hbcurl expone toda la API de curl

Negativas:
- La construcción del XML SOAP es manual (sin WSDL tooling)
- El parseo de respuesta CSV es manual
- Harbour es síncrono — la llamada SOAP bloquea la UI hasta recibir respuesta
