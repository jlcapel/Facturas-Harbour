# ADR-001: Elección de stack Harbour + HWGUI

## Estado
Aceptado

## Contexto
Necesitamos una aplicación de facturación VERI*FACTU multiplataforma. El proyecto .NET (Avalonia) ya existe y es funcional. Se nos pide clonar su funcionalidad usando Harbour.

## Decisión
Usar Harbour (core) como lenguaje y HWGUI como toolkit gráfico. Harbour es el compilador xBase moderno, open source, con soporte multi-plataforma. HWGUI es la librería GUI más madura para Harbour, con dos backends: Win32 API (Windows) y GTK2 (Linux).

## Consecuencias
Positivas:
- Código xBase único para Windows y Linux (HWGUI abstrae la diferencia)
- Harbour tiene SHA-256 built-in, SQLite3, JSON, XML via contribs
- Ejecutables autocontenidos, sin runtime externo
- Comunidad Harbour activa desde 1999

Negativas:
- Sin async/await — las llamadas SOAP a AEAT bloquean la UI
- Sin MVVM con binding reactivo — eventos directos de ventana
- Librerías menos pulidas que NuGet (hbhpdf es limitado frente a QuestPDF)
- 32-bit en algunas plataformas
