# ADR-008: Sin ORM — SQL directo con hbsqlit3

## Estado
Aceptado

## Contexto
El proyecto .NET usa EF Core (ORM) con LINQ. Harbour no tiene un ORM comparable.

## Decisión
Usar SQL directo via `hbsqlit3`. Las consultas se escriben como strings SQL. Harbour proporciona funciones para ejecutar consultas, recorrer resultados y mapear a variables/objetos.

## Consecuencias
Positivas:
- Control total sobre el SQL
- Sin magic, sin caché de entidades
- Consultas optimizables manualmente

Negativas:
- Más código boilerplate (mapeo manual de columnas a variables)
- Sin LINQ, sin comprobación en compile-time de las consultas
- Las relaciones hay que manejarlas manualmente (JOINs)
